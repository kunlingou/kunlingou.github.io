基础知识

什么是springmvc？

  • springmvc 是 spring 框架的一个模块。
  • springmvc 是基于mvc的web框架。
  • mvc 是一个设计模式。
  • dao，orm，jee，web，aop，core。
  • b/s系统下模型无法将数据填充到视图。

    框架原理

  • 前端控制器、处理器映射器、处理器适配器、视图解析器
  • 前端控制器DispacherServlet:接收用户请求,请求查找handler,请求执行handler，请求视图解析,视图渲染
  • 处理器映射器HandlerMapping:查找handler，返回执行链HandlerExecutionChain
  • 处理器适配器handlerAdapter:执行handler,返回ModelAndView
  • Handler 处理器：controler,返回ModelAndView
  • 视图解析器View resolver，返回view

需求
  • 商品列表
  • 修改界面

  • 提交

    开发
    • mapper
      • id查看商品信息
      • id更新商品信息
        (使用逆向工程生成的代码)
    • service
    • controller
      • 显示修改界面
      • 修改商品信息
    • @requestMapping:定义control方法对应的url
      • 窄化请求映射
      • 限制请求方法405

  • controller方法返回值

    • ModelAndView:需设置model、view

    • string：返回逻辑视图名

      • redirect重定向

      • 页面转发,request共享

    • void

      • request.getRequestDispacher(“”).forward(request,response)
      • response.senRedirect(“”);
      • 响应json

  • 参数绑定

    • 客户端请求key/value数据，参数绑定，绑定到control的形参上，converter(转换器)
    • 默认支持类型：
      • HttpServletRequest
      • HttpServletResponse
      • HttpSession
      • Model/ModelMap
        将model数据填充到request域

    • 简单类型@Requestparam

      • 不使用@Requestparam 传入的参数名和形参名一致
      • 使用@Requestparam
        • required:true是否必须传入
        • defaultValue:默认值

    • pojo绑定

    • 日期类型自定义参数绑定(处理器适配器)
      • 向处理器适配器注入参数绑定

  • springmvc 和 struts2 区别

    • springmvc基于方法开发，struts2基于类开发。
    • springmvc可以进行单例开发，struts无法使用单例开发。
    • struts2速度慢，在于使用了struts标签，如果使用struts，建议使用jstl。
